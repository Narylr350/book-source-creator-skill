# Legado 用户协助调试

## 使用约束

- 仅在以下情况进入调试模式：
  - 用户反馈“书源用不了了”
  - 导入失败
  - 搜索、详情、目录、正文任一链路失效
  - Legado 在登录、调试或阅读过程中崩溃
- 先判定失败阶段，再索取证据。
- 单次回复只索取当前阶段的最小证据包，不提前索要全部截图和日志。
- 可要求文本源码时，优先要求文本源码。
- 可在 Legado 内复现的问题，不先要求外部抓包或系统级日志。
- 不要只写内部术语，必须写明用户实际点击路径和界面里的中文菜单名。

## 官方入口依据

- 书源编辑页右上角三点菜单包含：
  - `登录`
  - `调试源`
- 调试页支持以下输入：
  - 搜索关键字
  - 详情页 URL
  - `++目录页URL`
  - `--正文页URL`
  - `标题::发现URL`
- 调试页右上角三点菜单包含：
  - `搜索源码`
  - `书籍源码`
  - `目录源码`
  - `正文源码`
  - `刷新发现`
- 关于页包含：
  - `崩溃日志`
  - `保存日志`

## 故障分流矩阵

| 失败阶段 | 典型症状 | 先让用户进入的位置 | 输入内容 | 最小证据包 |
| --- | --- | --- | --- | --- |
| 登录 | 登录后仍未生效、提示未登录、会员内容不可见 | 书源管理 -> 对应书源 -> 编辑页 -> 右上角三点 -> `登录` | 无 | `loginUrl/loginUi/loginCheckJs` 截图、登录页截图、登录成功后页面截图 |
| 导入 | JSON 无法导入、导入时报字段错误、导入后书源未出现 | 导入界面 | 无 | 导入报错截图、`book-source.json` 原文、必要时书源列表截图 |
| 搜索 | 无结果、报错、结果字段错乱 | 书源管理 -> 对应书源 -> 编辑页 -> 右上角三点 -> `调试源` | 搜索关键词 | 调试结果区截图、调试页右上角三点里的 `搜索源码` 原文、搜索规则截图 |
| 详情 | 能搜到书但详情打不开、字段空白、封面或简介异常 | 书源管理 -> 对应书源 -> 编辑页 -> 右上角三点 -> `调试源` | 详情页 URL | 调试结果区截图、调试页右上角三点里的 `书籍源码` 原文、详情规则截图 |
| 目录 | 详情正常但目录打不开、章节不全、顺序错误、目录分页失败 | 书源管理 -> 对应书源 -> 编辑页 -> 右上角三点 -> `调试源` | `++目录页URL` | 调试结果区截图、调试页右上角三点里的 `目录源码` 原文、目录规则截图 |
| 正文 | 正文空白、乱码、广告页、跳错页、仅首章正常 | 书源管理 -> 对应书源 -> 编辑页 -> 右上角三点 -> `调试源` | `--正文页URL` | 调试结果区截图、调试页右上角三点里的 `正文源码` 原文、至少两个异常章节 URL、正文规则截图 |
| 发现 | 发现页打不开、分类错乱 | 书源管理 -> 对应书源 -> 编辑页 -> 右上角三点 -> `调试源` | `标题::发现URL` | 调试结果区截图、调试页能看到的对应源码文本、发现规则截图 |
| 崩溃 | 调试页、登录页、阅读页闪退 | 我的 -> 关于 -> `崩溃日志` 或 `保存日志` | 无 | 崩溃日志截图或文本、`logs.zip`、Legado 版本号 |

## 规则截图字段矩阵

| 失败阶段 | 必要字段 |
| --- | --- |
| 登录 | `loginUrl`、`loginUi`、`loginCheckJs` |
| 搜索 | `searchUrl`、`bookList`、`name`、`bookUrl` |
| 详情 | `init`、`name`、`author`、`intro`、`tocUrl` |
| 目录 | `chapterList`、`chapterName`、`chapterUrl`、`nextTocUrl` |
| 正文 | `content`、`title`、`nextContentUrl`、`webJs`、`sourceRegex`、`replaceRegex` |
| 发现 | `exploreUrl`、`ruleExplore` 相关字段 |

## 回复结构

每次要求用户配合时，固定按以下顺序输出：

1. 从哪里进入
2. 点哪个菜单
3. 输入什么
4. 调试完成后回传哪几项证据
5. 若涉及登录、验证码、短信、扫码或人工确认，明确要求用户先手动完成

## 可直接发送的话术模板

### 登录问题

```md
先不要继续改规则，先确认登录态。

请在 Legado 里按这个路径操作：
1. 打开 `书源管理`
2. 找到这个书源，进入编辑页
3. 点右上角三点
4. 点 `登录`

如果登录过程需要扫码、验证码、短信或人工确认，请你先手动完成。

完成后把下面 3 项发给我：
1. `loginUrl`、`loginUi`、`loginCheckJs` 的截图
2. 登录页截图
3. 登录成功后落地页面的截图
```

## 调试阶段附加规则

- 若用户只说“打不开”“没内容”“报错了”，先把问题压缩成以下之一：
  - 搜索失败
  - 详情失败
  - 目录失败
  - 正文失败
  - 登录失败
  - 导入失败
  - 崩溃
- 若站点需要登录，不要在未确认登录态前继续分析正文或目录异常。
- 若正文只首章正常，默认优先怀疑：
  - `nextContentUrl`
  - `content` 提取范围
  - `webJs`
  - `sourceRegex`
  - `replaceRegex`
- 若目录不全，默认优先怀疑：
  - `chapterList`
  - `nextTocUrl`
  - 目录页分页入口
- 若搜索结果错乱，默认优先怀疑：
  - `bookList`
  - `bookUrl`
  - 搜索页是否为异步接口或多模板结果页
